# Полная спецификация KCore (финальная)

## 1. Общие сведения
- Расширение `.kcore`
- Кодировка UTF‑8
- Комментарии: `//` и `/* */`
- Конец инструкции `;`
- Блоки `{ ... }`
- Регистрозависимость

## 2. Типы данных

### 2.1 Базовые типы
| Тип | Размер | Знаковость |
|-----|--------|-------------|
| `байт` | 8 бит | знаковый/беззнаковый |
| `короткое` | 16 бит | знаковый/беззнаковый |
| `целое` | 32 бита | знаковый/беззнаковый |
| `длинное` | 64 бита | знаковый/беззнаковый |
| `дробное` | 32 бита (float) | всегда знаковый |
| `десятичное` | 64 бита (double) | всегда знаковый |
| `логическое` | 8 бит | `истина` / `ложь` |
| `битфлаговый` | 8 бит (8 флагов) | беззнаковый, доступ по битам |
| `ничто` | 32/64(как указатель) | тип для void / nullptr |

Модификаторы знаковости: `знаковый` (по умолчанию), `беззнаковый`. Применяются к целым и `битфлаговый`.

### 2.2 Производные типы
- **Указатель**: `<тип> *` (например, `целое* ptr`)
- **Ссылка** (только параметры функций): `<тип> &`
- **Статический массив**: `<тип> [] <имя>[<размер>];` или `<тип> [] <имя> = [<значения>];`
- **Структура**:
  ```
  структура <имя> {
      <тип> поле1;
      <тип> поле2 : <биты>;  // битовое поле
  };
  ```

## 3. Операторы (в порядке убывания приоритета)

| Приоритет   | Операторы                                                                         | Ассоциативность |
|-------------|-----------------------------------------------------------------------------------|-----------------|
| 1 (высший)  | `++` `--` (постфикс) `[ ]` `( )` `.`                                              | слева           |
| 2           | `++` `--` `+` `-` `!` `не` `~` `&` `*` (унарные)                                  | справа          |
| 3           | `**` (возведение в степень)                                                       | справа          |
| 4           | `*` `/` `%` `\` (целочисл. деление)                                               | слева           |
| 5           | `+` `-` (бинарные)                                                                | слева           |
| 6           | `<<` `>>` (сдвиги)                                                                | слева           |
| 7           | `<<<` `>>>` (вращения)                                                            | слева           |
| 8           | `<` `>` `<=` `>=`                                                                 | слева           |
| 9           | `==` `!=`                                                                         | слева           |
| 10          | `&` (побитовое И)                                                                 | слева           |
| 11          | `^` (побитовое XOR)                                                               | слева           |
| 12          | `\|` (побитовое ИЛИ)                                                              | слева           |
| 13          | `и` / `&&` (логическое И)                                                         | слева           |
| 14          | `или` / `\|\|` (логическое ИЛИ)                                                   | слева           |

## 4. Управляющие конструкции

### 4.1 Условный оператор
```
если (условие) { ... }
иначе если (условие) { ... }
иначе { ... }
```

### 4.2 Циклы
```
пока (условие) { ... }
для (инициализация; условие; шаг) { ... }
сначала { ... } потом-если (условие);
перебрать (тип элемент из массив) { ... }
```

### 4.3 Выбор
```
выбор (выражение) {
    метка значение1 { ... }
    метка значение2, метка значение3 { ... }
    по_умолчанию { ... }
}
```

### 4.4 Переходы
- `выход;` (break)
- `продолжить;` (continue)
- `перейти метка;` (goto)
- `вернуть [выражение];` (return)

## 5. Функции
- Объявление: `<тип> <имя>(<параметры>) { ... }`
- Параметры: `<тип> <имя>` или `<тип> & <имя>`
- Точка входа: `целое главная(целое argc, байт*[] argv) { ... }` (argv опционален)

## 6. Препроцессор (директивы с `#`)
- `#определить ИМЯ [значение]`
- `#включить "файл"`
- `#если (константное_выражение)` / `#иначе` / `#конец`
- `#еслиопределено(ИМЯ)` / `#еслинеопределено(ИМЯ)`
- `#ошибка "сообщение"`
- `#указание директива` (например, `#указание once`)

## 7. Встроенный ассемблер (KAsm)
- Поддерживается полный синтаксис KAsm (функциональный и операторный) с регистрами (`.eax`, `.rbx` и т.д.).
- Код, содержащий регистры, транслируется компилятором напрямую.
- Блок `ассемблер { ... }` вставляет сырой код без доступа к переменным.

# Примеры кода на KCore

## Пример 1. Рекурсивный факториал
```kcore
целое факториал(целое n) {
    если (n <= 1) {
        вернуть 1;
    }
    вернуть n * факториал(n - 1);
}

целое главная(целое argc, байт*[] argv) {
    целое x = 5;
    вернуть факториал(x);  // 120
}
```

## Пример 2. Структуры и битовые поля
```kcore
структура Флаги {
    байт a : 1;
    байт b : 2;
    целое c : 4;
};

целое главная() {
    Флаги f;
    f.a = 1;
    f.b = 3;
    f.c = 7;
    вернуть 0;
}
```

## Пример 3. Вращения и встроенный ассемблер
```kcore
целое главная() {
    .eax = 0x80000000;
    .eax <<<= 1;         // вращение влево
    .ebx = .eax >>> 1;   // вращение вправо
    вернуть .ebx;
}
```

## Пример 4. Цикл перебора (foreach) и выбор
```kcore
целое главная() {
    целое[] числа = [10, 20, 30];
    целое сумма = 0;
    перебрать (целое x из числа) {
        сумма = сумма + x;
    }
    
    выбор (сумма) {
        метка 60 { сумма = 100; выход; }
        по_умолчанию { сумма = 0; }
    }
    
    вернуть сумма;
}
```

## Пример 5. Указатели и ссылки
```kcore
ничто обмен(целое& a, целое& b) {
    целое t = a;
    a = b;
    b = t;
}

целое главная() {
    целое x = 10, y = 20;
    обмен(x, y);   // x=20, y=10
    целое* ptr = &x;
    *ptr = 100;
    вернуть x;     // 100
}
```
