# KPL

**KPL (Kurluk Programming Language)** — семейство языков:
- **K** — полноценный язык с ООП (замена C++/Java)  
- **KCore** — низкоуровневое ядро (замена C)  
- **KAsm** — ассемблер для целевой архитектуры  
- **KBC** — байт-код для виртуальной машины  

---

# Спецификация KCore

## 1. Общие правила
- Расширение файла: `.kcore`
- Кодировка: UTF‑8
- Комментарии: `//` (до конца строки) и `/* ... */` (многострочные)
- Конец инструкции: **пробел + `\>`** (пробел обязателен)
- Пустая инструкция запрещена
- Регистрозависимость: `целое` ≠ `Целое`
- Блоки кода: всегда `{ ... }`

## 2. Типы данных
- `байт` (8 бит)
- `короткое` (16 бит)
- `целое` (32 бита)
- `длинное` (64 бита)
- `дробное` (float, 32 бита)
- `десятичное` (double, 64 бита)
- `логическое` (`истина`, `ложь`)
- `ничто` (void / nullptr)

Указатели: `<тип> *`  
Ссылки (только параметры функций): `<тип> &`

## 3. Переменные, константы, присваивание
- Объявление переменной: `<тип> <имя> [ = <выражение> ] \>`
- Объявление константы: `константа <имя> = <выражение> \>`
- Присваивание: `<имя> = <выражение> \>`

## 4. Массивы (статические)
- С размером: `<тип> [] <имя> [ <константное_выражение> ] \>`
- С инициализацией: `<тип> [] <имя> = [ <выражение> , ... ] \>`
- Доступ: `<имя> [ <выражение> ]`

## 5. Функции
- Объявление: `<тип> <имя> ( <параметры> ) { ... }`
- Параметры: `<тип> <имя>` или `<тип> & <имя>`
- Возврат: `вернуть <выражение> \>`  
- Для `ничто`-функций: `вернуть \>`  
- Для возврата нулевого указателя: `вернуть ничто \>`

## 6. Операторы (приоритет от высшего к низшему, ассоциативность)

| Приоритет | Операторы | Ассоциативность |
|-----------|-----------|------------------|
| 1 | `++` `--` (постфиксные), `[ ]` `( )` | слева |
| 2 | `++` `--` `+` `-` `~` `!` `не` `*` `&` (унарные) | справа |
| 3 | `**` | справа |
| 4 | `*` `/` `%` `\` (целочисл. деление) | слева |
| 5 | `+` `-` (бинарные) | слева |
| 6 | `<<` `>>` | слева |
| 7 | `<` `>` `<=` `>=` | слева |
| 8 | `==` `!=` | слева |
| 9 | `&` (побитовое И) | слева |
| 10 | `^` (побитовое XOR) | слева |
| 11 | `|` (побитовое ИЛИ) | слева |
| 12 | `и` (логическое И) | слева |
| 13 | `или` (логическое ИЛИ) | слева |
| 14 | `? :` | справа |
| 15 | `=` `+=` `-=` `*=` `/=` `%=` `\=` `**=` `&=` `|=` `^=` `<<=` `>>=` | справа |

## 7. Управляющие конструкции
- `если ( <условие> ) { ... } [ иначе если ( <условие> ) { ... } ] [ иначе { ... } ]`
- `пока ( <условие> ) { ... }`
- `для ( <инициализация> ; <условие> ; <шаг> ) { ... }`
- `сначала { ... } потом-если ( <условие> ) \>`
- `выход \>`
- `продолжить \>`

## 8. Идентификаторы
- Символы Unicode (кириллица, латиница, цифры, `_`, эмодзи), кроме пробелов и спецсимволов.
- Не начинаются с цифры.

## 9. Литералы
- Целые: `123`, `1_000_000`, `0o777`, `0xFF`, `0b1010`
- С плавающей точкой: `3.14`, `1e-3`, `-0.001`
- Строки: `"..."`, escape: `\\`, `\"`, `\'`, `\нс` (новая строка), `\то` (табуляция), `\з` и т.п.
- Логические: `истина`, `ложь`
- Нулевой указатель: `ничто`

## 10. Ввод/вывод
Отсутствует. Предоставляется стандартной библиотекой.
